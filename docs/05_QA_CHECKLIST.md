# 05. Чек-лист приемки и тестирования

## 1. Архитектура

- [ ] `domain` не зависит от `data` и `presentation`
- [ ] `presentation` не ходит в Supabase напрямую
- [ ] `data` скрывает SDK-детали от `domain`
- [ ] валидация email/password вынесена в `domain`
- [ ] ProductCard — отдельный компонент

## 2. Register Account

- [ ] экран соответствует макету
- [ ] email проверяется локально
- [ ] некорректный email показывает диалог
- [ ] пароль можно показать/скрыть
- [ ] иконка меняется вместе с состоянием
- [ ] при успешной регистрации происходит переход на `Sign In`
- [ ] во время запроса видна загрузка

## 3. Sign In

- [ ] проверка пустых полей работает
- [ ] ошибка пустых полей показывается в диалоге
- [ ] некорректный email показывает диалог
- [ ] пароль можно показать/скрыть
- [ ] `Восстановить` ведет на `Forgot Password`
- [ ] `Создать пользователя` ведет на `Register Account`
- [ ] успешный логин открывает main graph

## 4. Forgot Password

- [ ] корректный email отправляет recovery request
- [ ] после успеха показывается диалог
- [ ] тап по диалогу ведет на `OTP Verification`
- [ ] `Назад` возвращает на `Sign In`

## 5. Verification

- [ ] OTP вводится по ячейкам
- [ ] таймер стартует с 01:00
- [ ] повторный запрос невозможен до конца таймера
- [ ] после истечения таймера повторный запрос доступен
- [ ] неверный код красит все ячейки в error-state
- [ ] верный код открывает `Create New Password`

## 6. Create New Password

- [ ] проверка совпадения паролей работает
- [ ] слишком короткий пароль блокируется
- [ ] успешное изменение ведет на `Sign In`
- [ ] во время запроса видна загрузка

## 7. Home

- [ ] акции читаются с сервера
- [ ] товары/секции отображаются
- [ ] при ошибке загрузки показано корректное состояние
- [ ] нижняя навигация отображается корректно

## 8. Catalog

- [ ] товары читаются с сервера
- [ ] карточки товара отображаются корректно
- [ ] состояние избранного синхронизировано
- [ ] empty-state отображается корректно при пустом каталоге

## 9. Favorite

- [ ] загружаются только товары текущего пользователя
- [ ] добавление в избранное сохраняется на сервере
- [ ] удаление из избранного синхронизируется с сервером
- [ ] empty-state отображается при отсутствии избранного

## 10. Side Menu

- [ ] боковое меню соответствует макету
- [ ] `Выйти` вызывает logout
- [ ] после logout пользователь попадает на `Sign In`
- [ ] старая сессия не используется повторно

## 11. Profile / Edit Profile

- [ ] данные профиля читаются с сервера
- [ ] изменение полей сохраняется на сервере
- [ ] диалог выбора фото содержит `Камера` и `Галерея`
- [ ] выбор из галереи работает
- [ ] съемка камерой работает
- [ ] фото загружается в Storage и отражается в профиле
- [ ] штрихкод строится из user id
- [ ] тап по штрихкоду открывает `Loyalty Card`

## 12. Loyalty Card

- [ ] экран открывается из профиля
- [ ] штрихкод отображается корректно
- [ ] при повороте/перерисовке экран сохраняет корректное состояние

## 13. UX и стабильность

- [ ] каждая сетевая операция показывает loading
- [ ] кнопки submit не отправляют форму многократно при быстром тапе
- [ ] серверные ошибки не приводят к падению приложения
- [ ] пустые ответы сервера не приводят к crash
- [ ] изображения не ломают layout при медленной сети

## 14. Backend / Security

- [ ] используется только anon/publishable key
- [ ] service role не попал в клиент
- [ ] RLS включен для пользовательских таблиц
- [ ] `profiles` изолирован по `auth.uid()`
- [ ] `favourite` изолирован по `auth.uid()`
