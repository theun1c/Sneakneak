# Sneakneak - контекст проекта

Мобильное приложение для интернет-магазина кроссовок, которое включает в себя полный цикл взаимодействия с пользователем: от регистрации и авторизации до просмотра каталога, управления избранным и редактирования профиля.

---

### Общая идея системы

#### Цели и задачи системы
- Аутентификация и управление доступом: регистрация, вход, восстановление пароля через OTP-коды, защита пользовательских данных.
- Просмотр и навигация по каталогу: отображение товаров с возможностью фильтрации и поиска, детальная информация о каждом товаре.
- Управление избранным: добавление и удаление товаров из списка избранного с синхронизацией на сервере.
- Управление профилем: просмотр и редактирование личных данных, смена фото профиля (камера/галерея), отображение штрих-кода карты лояльности.
- Акции и специальные предложения: отображение актуальных акций на главном экране, загружаемых с сервера.
- Боковое меню: навигация по разделам приложения, возможность выхода из аккаунта.

---

#### Компоненты системы
- Клиентское мобильное приложение: Android-приложение, через которое пользователи взаимодействуют с системой.
- Серверная часть (Supabase): аутентификация пользователей, база данных для хранения информации о товарах, пользователях, избранном и акциях, файловое хранилище для изображений профиля и товаров.
- Аппаратные возможности устройства: интеграция с камерой и галереей для выбора фото профиля.

---

### Технический стек

Проект разрабатывается под Android (minSdk 30+), поэтому для него были выбраны следующие технологии:

- Kotlin (язык разработки)
- Jetpack Compose (UI)
- Clean Architecture + MVVM (архитектура приложения)
- Supabase (бэкенд: аутентификация, база данных, файловое хранилище)
- Coroutines + Flow (асинхронность и потоки данных)
- Jetpack Navigation Component (навигация)
- Coil/Glide (загрузка изображений)
- CameraX (работа с камерой)
- ZXing (генерация штрих-кода)

### Архитектура

Разрабатываемое мобильное приложение строится на многослойной архитектуре с четким разделением ответственности:

- **Domain слой (независимый):**
  - Модели данных (User, Product, Promotion)
  - Интерфейсы репозиториев
  - Сценарии использования (use cases)
  - Не содержит зависимостей от других слоев и фреймворков

- **Data слой:**
  - Реализация репозиториев
  - Источники данных (Supabase API, локальное хранилище)
  - Мапперы для преобразования DTO в доменные модели
  - Работа с сетью и кэшированием

- **Presentation слой:**
  - ViewModel (управление состоянием экранов)
  - Composables (UI компоненты)
  - Навигация между экранами
  - Разделение на фичи (auth, home, catalog, profile, favorites)

#### Принципы архитектуры:
- Строгое соблюдение направленности зависимостей (внутрь Domain)
- Независимость Domain слоя от фреймворков и внешних библиотек
- Изменение бизнес-логики или представления одного экрана не влечет за собой изменение других экранов
- Каждый слой находится в отдельном пакете/модуле

### Основные функциональные блоки

1. **Модуль аутентификации** (5 экранов):
   - Register Account
   - Sign In
   - Forgot Password
   - Verification (OTP)
   - Create New Password

2. **Модуль главного экрана и каталога** (3 экрана):
   - Home (акции, приветствие)
   - Catalog (список товаров)
   - Favorite (избранное)

3. **Модуль профиля и навигации** (3 экрана):
   - Profile (личные данные, штрих-код)
   - Loyalty Card (карта лояльности)
   - Side Menu (боковое меню)

4. **Аппаратная интеграция**:
   - Выбор фото из галереи
   - Съемка фото через камеру

5. **Клиент-серверное взаимодействие**:
   - Регистрация/авторизация
   - Загрузка товаров и акций
   - Синхронизация избранного
   - Управление профилем
   - Восстановление пароля
   - Деавторизация
