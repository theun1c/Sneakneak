# 👟 Мобильное приложение Sneakneak

Данный проект представляет собой клиентскую часть мобильного приложения для интернет-магазина кроссовок, разработанную под Android (minSdk 30+). Приложение включает в себя полный цикл взаимодействия с пользователем: регистрацию, авторизацию, восстановление доступа, просмотр каталога и управление профилем. 🚀

## 📋 Функциональные требования

Проект разработан в соответствии с техническим заданием, охватывающим три основных блока: верстка экранов, клиент-серверное взаимодействие и работа с аппаратными функциями устройства.

### 1. 🔐 Экраны аутентификации и профиля (Верстка и навигация)

*   **📝 Экран "Register Account" (Регистрация):**
    *    Валидация email по строгому паттерну (имя и домен: латиница нижний регистр + цифры; старший домен: только буквы >2 символов). Ошибка выводится в диалоговом окне.
    *    Отображение/скрытие пароля с изменением иконки глаза.
    *    Отправка запроса на сервер для регистрации.
    *    Переход на экран "Sign In" при успешной регистрации или по кнопке "Войти".
*   **🔑 Экран "Sign In" (Вход):**
    *    Валидация email (аналогично регистрации).
    *    Проверка полей на пустоту.
    *    Отображение/скрытие пароля.
    *    Отправка запроса на сервер для авторизации.
    *    Навигация: "Восстановить" -> "Forgot Password", "Создать пользователя" -> "Register Account".
*   **🔄 Экран "Forgot Password" (Восстановление пароля):**
    *    Валидация email.
    *    Отправка запроса на сервер для получения кода.
    *    Отображение диалогового окна при успехе с переходом на экран "OTP Verification".
    *    Кнопка "Назад" ведет на экран "Sign In".
*   **🔢 Экран "Verification" (Ввод кода):**
    *    Таймер обратного отсчета (01:00) с возможностью повторной отправки запроса по его истечению.
    *    Визуальная индикация ошибки (все поля становятся красными) при неверном коде.
    *    Отправка кода на сервер для верификации.
*   **🔐 Экран "Create New Password" (Новый пароль):**
    *    Проверка совпадения паролей.
    *    Отправка нового пароля на сервер.
    *    Переход на экран "Sign In" после сохранения.
*   **👤 Экран "Profile" (Профиль пользователя):**
    *    Загрузка и отображение данных пользователя с сервера.
    *    Редактирование профиля через модальное окно "Edit Profile" с отправкой изменений на сервер.
    *    Смена фото профиля (камера или галерея).
    *    Генерация и отображение штрих-кода на основе ID пользователя.
    *    Переход на экран "Loyalty Card" по нажатию на штрих-код.
*   **💳 Экран "Loyalty Card":** Отображение карты лояльности (дизайн согласно макету).

### 2. 🏠 Основные экраны приложения и навигация

*   **🏡 Экран "Home" (Главная):**
    *    Отображение приветствия и сведений об акциях, загруженных с сервера.
    *    Наличие нижнего навигационного меню.
    *    UI-компонент "Карточка товара" для отображения в каталоге.
*   **📋 Экран "Catalog" (Каталог):**
    *    Загрузка списка товаров с сервера.
    *    Возможность добавления/удаления товара в избранное (отправка запроса на сервер).
*   **⭐ Экран "Favorite" (Избранное):**
    *    Загрузка списка избранных товаров для конкретного пользователя с сервера.
*   **📱 Side Menu (Боковое меню):**
    *    Реализация бокового меню согласно макету.
    *    Функция "Выйти": отправка запроса деавторизации на сервер и переход на экран "Sign In".

### 3. 🛠️ Технические требования и архитектура

*   **🏗️ Архитектура:** Чистая многослойная архитектура с разделением на слои:
    *   **DATA:** Работа с источниками данных (Supabase API, локальное хранилище).
    *   **DOMAIN:** Бизнес-логика, модели данных, интерфейсы репозиториев. **Не зависит от других слоев.**
    *   **PRESENTATION:** ViewModel, UI (экраны, адаптеры), навигация.
    *    Изменения в одном слое не должны влиять на другие.
*   **⚙️ Бэкенд:** Использование **Supabase** для аутентификации, базы данных и хранения файлов.
*   **🌐 Сеть:**
    *    Регистрация, авторизация, восстановление пароля, изменение данных.
    *    Загрузка акций, каталога, избранного.
    *    Индикация загрузки (прогресс-бар) при обмене данными с сервером.
*   **📱 Аппаратные возможности:**
    *    Выбор фото профиля из галереи.
    *    Съемка фото профиля с помощью камеры.
*   **🎨 UI/UX:**
    *    Использование единой темы приложения (цвета, шрифты) в соответствии с руководством по стилю.
    *    Поддержка Android 11+ (API 30+).

## 🛠️ Стек технологий

-  Язык: **Kotlin**
-  UI: **Jetpack Compose**
-  Асинхронность: **Coroutines, Flow**
-  Архитектура: **Clean Architecture, MVVM**
-  Бэкенд: **Supabase**
